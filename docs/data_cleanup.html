How to Keep Your Data Clean

Clearing Intermediate Results

It is recommended to use CryoSPARC's built-in feature to clear intermediate results as often as possible. This method helps manage storage more efficiently by clearing data on a job-by-job basis instead of having to clear the entire project at once.

You can select and clear multiple jobs at once-typically around 10-15 jobs-by Ctrl-clicking them. This is especially beneficial for refinement jobs, which tend to generate a large amount of intermediate data. Regularly clearing these results can help free up storage space and maintain optimal system performance.

In Relion you can use the soft clean function to clear intermediate results.

Deleting Unproductive Jobs

Removing jobs that did not yield useful results is a crucial practice for keeping your storage clean and your project organized. Deleting these unproductive jobs frees up valuable space and minimizes clutter from unnecessary data.

Keep in mind that in Relion, deleted jobs are saved in the Trash folder, so you need to empty the Trash folder regularly to free up storage space.

Moreover, by eliminating these jobs, you simplify your processing tree, making it easier to navigate and manage your workflow in future steps.

Clearing Extract Jobs

When the extracted particles are no longer needed-such as after completing a project or following a restacking or re-extraction-it is beneficial to clear the extract jobs. This process deletes the actual particle data, which can be quite large, while preserving the job skeleton, including the settings.

By clearing these jobs, you free up storage space without losing the ability to rerun the extraction if needed. This is quite important to keep storage usage as low as possible.

Deleting imported volumens

If you have imported volumes that are no longer needed, it is a good practice to delete them. This helps in managing storage effectively and ensures that only relevant data is retained in your project. If you imported volumes in cryosparc via drag & drop, you will find them in the imports folder of your project. Please clean this folder regularly to avoid unnecessary storage usage.

Deleting projects

  • To delete a cryoSPARC project you can use the built-in feature in the cryoSPARC web interface. This does not delete the corresponding project folder on the filesystem, so you will have to delete it manually. To do this, navigate to the project folder in the terminal and use the command: 
    rm -rf $project
    where $project is the name of the project you want to delete.
  • To delete a Relion project, you have to delete the project folder manually. Navigate to the project folder in the terminal and use the command: 
    rm -rf $project
    where $project is the name of the project you want to delete.
  • To delete a general project folder you can just delete the folder under /sbdata/projects/$username/$project using the command: 
    rm -rf $project
    where $project is the name of the project you want to delete. Before you do this delete the cryoSPARC project before to remove it from the database. Deleting a project will not delete the corresponding rawdata, radata folders and files inside the project folder are just symlinks to the original files.
