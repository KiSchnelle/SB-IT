Using CatchEM

Load the CatchEM environment

CatchEM is provided in a dedicated conda environment. Start from a compute node (e.g., an interactive Slurm session) and activate it with the Spack-installed miniforge3:

spack load miniforge3
source $(spack location -i miniforge3)/bin/activate
conda activate catchem

If your shell sets $MINIFORGE3_ROOT after loading, you can also run source $MINIFORGE3_ROOT/bin/activate before conda activate catchem. Keep long-running shells safe from disconnects with screen, and start your work inside an interactive job as shown in the Slurm interactive session notes.

Example interactive session

Launch an interactive job, load CatchEM, and keep it attached to a screen session so you can reconnect later if needed:

Optional but recommended (keeps the session alive if disconnected):

screen -S catchem-run

Then request an interactive Slurm session with appropriate resources for your dataset and model. Adjust the partition, CPUs, memory, and GPUs as needed:

srun --partition=p.cryo --ntasks=1 --cpus-per-task=30 --mem=300G --gres=gpu:4 --pty bash

Once on the node:

spack load miniforge3
source $(spack location -i miniforge3)/bin/activate
conda activate catchem

Run your CatchEM commands inside that shell. Detach with Ctrl+A then D, and reattach with screen -r catchem-run. See the screen guide for more tips.

Common commands

Replace the placeholders with your values: $motioncorrected_folder is your input data path, and $cpus_reserved should match what you requested (in an interactive Slurm shell you can use $SLURM_CPUS_PER_TASK if set).

Single particles

catchEM inference detection-to-particles $motioncorrected_folder --number-of-cpus $cpus_reserved --output-folder results_single --confidence 0.3

--confidence defaults to 0.5; 0.3 often yields better single-particle results.

Liposomes

catchEM inference detection-to-particles $motioncorrected_folder --number-of-cpus $cpus_reserved --image-operations-pipeline R --resize-image-size 1280 --inference-target liposomes

Add --output-folder if you want a custom destination for liposome outputs.

Use -h with any level for built-in help, for example catchEM -h, catchEM inference -h, or catchEM inference detection-to-particles -h.

Batch submission (sbatch) example

Submit an unattended job when you do not need to stay attached. Adjust partition, time, CPUs, GPUs, and memory to match your dataset. Ensure the logs folder exists or change the log paths.

#!/bin/bash
#SBATCH --job-name=catchem_detect
#SBATCH --partition=p.cryo
#SBATCH --ntasks=1
#SBATCH --cpus-per-task=30
#SBATCH --mem=300G
#SBATCH --gres=gpu:4
#SBATCH --output=logs/%x_%j.out
#SBATCH --error=logs/%x_%j.err

spack load miniforge3
source $(spack location -i miniforge3)/bin/activate
conda activate catchem

motioncorrected_folder=/path/to/motioncorrected
output_folder=/path/to/results

catchEM inference detection-to-particles $motioncorrected_folder \
  --number-of-cpus $SLURM_CPUS_PER_TASK \
  --output-folder $output_folder \
  --confidence 0.3

Remove --gres=gpu:4 if you do not need GPUs. Tune --confidence, CPUs, and memory to your model and data size.
