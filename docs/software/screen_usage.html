Using screen for long-running shells

Keep interactive Slurm sessions alive if you disconnect.

Why use screen?

screen lets you detach from a shell and reconnect later, which is ideal for long interactive Slurm jobs or unstable network connections. Start it on a compute node, not the headnode.

Quick start

Run these after you have an interactive job (see Slurm interactive sessions):

# Start a named session
screen -S myrun

# Do your work here
python script.py

# Detach (keep it running)
Ctrl+A then D

# List sessions
screen -ls

# Reattach
screen -r myrun

# If attached elsewhere
screen -d -r myrun

# End the session when done
exit

Shortcut: memorize Ctrl+A then D to detach and screen -r name to return.

Tips for interactive jobs

  • Name sessions meaningfully: screen -S catchem-run, screen -S preprocessing.
  • Keep logs inside the session: python script.py > logs/run.out 2>&1.
  • To scroll output, use copy mode: Ctrl+A then [, navigate with arrows, exit with Esc.
  • If a session looks frozen, detach with Ctrl+A then D and reattach.
  • Close unused sessions with exit inside them so they stop using resources.
