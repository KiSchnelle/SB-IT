Guide to Creating a New Project in Our Working Environment

Important Note on CryoSPARC Projects

If multiple users are collecting data for the same project, each user should create their own CryoSPARC project, even if the overall project is the same. This ensures that data is properly managed and avoids conflicts when users are leaving.

Project Folder Structure

Each user has a dedicated folder under:

/sbdata/projects/$username

When a session is transferred a project folder is automatically generated within the user's folder based on the rawdata session name if the project is not already existing. The session naming follows this convention: 

$username/YYYY-MM-DD_$username_$project_$comment

Where:

  • YYYY-MM-DD is the date in the format year-month-day (e.g., 2024-12-26).
  • $username is the user's name.
  • $project is the chosen project name.
  • $comment is an optional descriptive comment.

Session Collection and Project Naming

  • When collecting a session, the project name can be chosen the First Time.
  • For subsequent sessions, the project name must match exactly (case-sensitive) to ensure proper sorting into the same folder.
  • The CryoSPARC project should be named the same and should be placed in the cryosparc folder under either the cryo or negative_stain directory, depending on the data type.

Collecting Data for Other Users

When collecting data for other users the session username should be not the user who actually collects the data but the user who should receive the data.
